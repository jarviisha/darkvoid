**Input**: Coverage analysis ‚Äî handler 67.2%, service 50.7%, repository 0.0%

## Summary

Add meaningful tests for `internal/feature/post/handler/` and `internal/feature/post/service/`
targeting real behavioral gaps: the completely untested `ToggleCommentLike` and hashtag handlers,
the `LikeService.Toggle` and `CommentLikeService.Toggle` state machines (both 0%), the
`HashtagService` methods (all 0%), and the `CommentService` enrichment helpers (6‚Äì14%). Repository
tests are descoped: constitution Principle II prohibits mocking the database, and each repository
method is a 3-line sqlc pass-through where the only interesting logic (`MapDBError`) lives in a
shared infrastructure function. Coverage is a side-effect ‚Äî every new test asserts behavior a
regression could break.
